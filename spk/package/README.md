# SyrvisCore - Installation Package

This directory contains the Python package for SyrvisCore that will be installed on your Synology NAS.

## What's Included

- **syrviscore/** - Main Python package with CLI tools
- **pyproject.toml** - Package configuration and dependencies
- **build/config.yaml** - Docker image version definitions
- **LICENSE** - MIT License
- **README.md** - This file

## Installation

This package is automatically installed during SPK installation. You don't need to manually install it.

The installation process:
1. Creates a Python virtual environment at `/volumeX/syrviscore/cli/venv`
2. Installs this package into the virtual environment
3. Creates a symlink at `/usr/local/bin/syrvis` for easy CLI access

## CLI Usage

After installation, you can use the `syrvis` command from anywhere:

```bash
# Check version
syrvis --version

# Generate docker-compose.yaml
syrvis compose generate

# Start all services
syrvis start

# Check status
syrvis status

# View logs
syrvis logs traefik
syrvis logs portainer

# Stop all services
syrvis stop
```

## Configuration

Configuration is stored in `.env` file at the installation directory (e.g., `/volume5/syrviscore/.env`).

This file is automatically generated during installation based on your wizard inputs.

## Directory Structure

After installation:
```
/volumeX/syrviscore/
├── cli/                    # Python virtual environment
│   └── venv/
│       └── bin/syrvis      # CLI executable
├── data/                   # Persistent data
│   ├── traefik/
│   │   ├── acme.json       # SSL certificates
│   │   └── config/         # Dynamic config
│   ├── portainer/          # Portainer data
│   └── cloudflared/        # Cloudflare tunnel data
├── .env                    # Configuration
├── .env.template           # Configuration template
├── .syrviscore-manifest.json  # Installation metadata
├── docker-compose.yaml     # Generated compose file
└── README.md               # Installation guide
```
